What is it?

Flexbox is a set of CSS properties that provide a flexible way to lay out content.

There are three important points to keep in mind:

Flex Container:

  • Sets the context for the flex box layout.
  • Contains flex items, the actual elements that you layout using flex box.
  • Can be any block-level or inline element.

Flex Items:

  • Every direct child of a flex container.
  • There can be any number of flex items inside a flex container.

Flexbox Axes:

  • Main axes — where primary flex items are laid out, the main start/main end direction is left–to–right.
  • Cross axis — the main start/main end direction is top–to–bottom.

Creating a Flex Container!

Return to Index

The flex container is used with display: flex; — this creates a block–level flex container.

  • Any element outside a flex container is not affected by the flex box layout.
  • Only children of a flex container automatically become flex items.
  • Some flexbox properties apply to the flex container only, while some may apply only to the flex items.

Working with Flex Directions

Return to Index

The flex direction property applies to the flex container only.

The default value for flex direction is row, using display: flex.

To reverse the flex direction in a row, use flex–direction: row–reverse, as shown above.

Flex direction also applies to making a column. In your code, use flex-direction: column.

Reverse a column by using flex-direction: column-reverse.

Working with Flex–Wrap

Return to index

By default, flex items will try to fit onto one line. You can change this property and allow the items to wrap onto a single line or multiple lines as needed.

The default property is flex–wrap: (nowrap). All flex items are on one line. *Make this screen narrow, and the following images will NOT wrap to the next line.

flex–wrap: wrap will wrap flex–items onto multiple lines.

flex–wrap: wrap–reverse will wrap flex–items onto multiple lines in reverse order.

Justify–content and Flex

Return to index

These properties can control the position of the flex box as a whole, in relation to the flexbox container.

There is the following:

  • flex–start
  • flex–end
  • center
  • space–between
  • space–around

flex–start moves content to the beginning of the page. Which is actually default behavior.

flex–end moves content to the end of the page.

space–between creates equal spacing between flex–items. Items are evenly spaced on the line.

space–around creates items evenly spaced within the line, with equal space around them. Note the visual differences between space–between and space–around. Not quite the same.

Flex–grow & Flex–shrink

Return to index

Flex–grow property applies to flex–items only.

This defines the ability for a flex–item to grow, if needed. It controls what amount of the available space inside the flex container the item should take up.

Example below in center is using flex–grow: 2. This value is taking up twice as much space as the others. The higher the grow value, the more an item grows relative to the other items. *Negative numbers are invalid.

flex–shrink allows for a flex–item to shrink, if needed.

In example below, the center paragraph is using flex–shrink: 1. This flex–item shrinks in relative to the rest of the flex items in the flex container.
